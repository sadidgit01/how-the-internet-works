# How the Internet Works

**Every time you search, log in, or load a page, your request travels through
roughly 17 real systems before you ever see a result.** This is that journey —
broken down layer by layer, from the moment you hit Enter to the moment
pixels appear on your screen.

DNS resolution, TCP and TLS handshakes, firewalls, CDNs, load balancers, API
gateways, reverse proxies, service meshes, app servers, caches, databases,
message queues, background workers, and the response path back — each one
explained with what it is, how it works, and a real-world example.

Then, to make it concrete: the exact same 17 steps, rebuilt around one
specific request — **Netflix, searching for "Stranger Things"** — with real
system behavior, real numbers, and real code.
